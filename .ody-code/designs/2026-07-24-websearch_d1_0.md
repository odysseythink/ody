# WebSearch 重构 D1.0 详细设计

## Scope In / Scope Out

### Scope In [C:USER]
- 完全删除 Rust 端现有 web search 运行时路径：hosted `web_search`、standalone `web/run` 扩展、`ody-api::SearchClient` / `alpha/search` 端点、provider/model 层 `web_search` 与 `supports_search_tool` 能力门控、`web_search_mode` 配置及 `Feature::StandaloneWebSearch` / `Feature::WebSearchCached` / `Feature::WebSearchRequest`。
- 新建 `ody-web-search` crate，对齐 TS `ody-code`：
  - `WebSearchProvider` trait / `WebSearchResult` / `WebSearchOptions`；
  - `WebSearchProviderRegistry` + `FallbackWebSearchProvider`；
  - 12 个 provider 实现：DuckDuckGo、Bing、SerpApi、SearchApi、Serper、Baidu、Serply、SearXNG、Tavily、Exa、Perplexity、Moonshot。
- 新增 `services.webSearch` 配置 schema（`primary` + 可选 `secondary`，每个 provider/apiKey/timeoutMs/options）。
- `app-server` 解析配置并创建 provider，通过 `ToolContributor` 注入 `WebSearchTool` 到 `core`。
- `core` 中实现 `WebSearchTool`：工具名 `WebSearch`，输入 `{query, limit, include_content}`，输出纯文本 `Title/Date/URL/Snippet/Content`。
- TUI 工具结果 chip 渲染：按结果条数显示 `N results` / `no results` / `web result`。
- 单测：provider 单元测、registry/fallback 单元测、配置解析单测、工具执行单测、TS-Rust parity 测试。

### Scope Out [C:USER]
- 不保留 Responses API hosted `web_search` 原生工具路径。
- 不保留 `web/run` namespaced extension 工具路径。
- 不保留 `OpenPage` / `FindInPage` 语义；新 `WebSearchTool` 只支持 `query` 搜索。
- 本次不实现 OAuth 刷新链路（`resolveOAuthTokenProvider` 后续补齐）。
- 不修改 LLM 协议层对通用工具 schema 的处理，只修改 web search 工具自身的 schema。
- 不引入 keyring 或环境变量注入 API key；配置中 `apiKey` 为明文字符串，与 TS 一致。

## Architecture / Design

### 总体架构 [C:USER]

```
┌─────────────────────────────────────────────────────────────┐
│  config.toml                                                │
│  [services.webSearch]                                       │
│  primary = { provider = "duckduckgo" }                      │
│  secondary = { provider = "bing", apiKey = "..." }          │
└──────────────────────┬────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  ody-config                                                 │
│  parse `services.webSearch` → `WebSearchConfig`             │
└──────────────────────┬────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  ody-app-server                                             │
│  resolveWebSearchRuntime(config) → Option<WebSearchProvider> │
│  if Some, inject WebSearchTool via ToolContributor           │
└──────────────────────┬────────────────────────────────────────┘
                       │ Arc<dyn WebSearchProvider>
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  ody-core                                                   │
│  WebSearchTool.execute() → provider.search(query, opts)     │
│  format results → plain text ToolOutput                     │
└──────────────────────┬────────────────────────────────────────┘
                       │ ToolOutput
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  ody-tui                                                    │
│  chip renderer: N results / no results / web result          │
└─────────────────────────────────────────────────────────────┘
```

### Crate 边界 [C:USER]
- `ody-web-search`：trait、注册表、fallback、错误分类、配置类型、12 个 provider 实现。不依赖 `ody-core` / `ody-app-server`，只依赖 `ody-protocol` / `serde` / `reqwest` / `tracing` 等基础库。
- `ody-config`：解析 `services.webSearch` 为 `ody_web_search::WebSearchConfig`。
- `ody-app-server`：根据 `Config` 创建 `WebSearchProvider` 并注入；依赖 `ody-web-search`。
- `ody-core`：实现 `WebSearchTool` 并接入工具规划；通过 trait 调用 provider，不直接解析配置。
- `ody-tui`：纯文本 chip 渲染。

## Assumptions & Unverified Items

| # | Assumption | Confidence | Impact if wrong | How to verify |
|---|---|---|---|---|
| 1 | `ody-api::SearchClient` 和 `alpha/search` 端点仅被 web search 使用，删除后无其他使用方。 | medium | 编译失败或误删功能 | 删除前全 workspace grep `SearchClient`、`alpha/search` |
| 2 | `WebSearchItem` / `WebSearchAction` 协议字段未在持久化 thread history 中被读取后重新渲染；删除后旧历史可降级为纯文本。 | medium | 旧 thread 无法打开或渲染异常 | 检查 `app-server-protocol/src/protocol/thread_history.rs` 和 `core/src/context_manager/history.rs` 的序列化/反序列化路径 |
| 3 | `ody-app-server` 现有的 `ExtensionRegistry` / `ToolContributor` 机制足以承载动态工具注册（无 provider 时不注册 `WebSearchTool`）。 | high | 需要额外工具注册机制，延期 | 检查 `app-server/src/extensions.rs` 和 `ody-extension-api` 的 ToolContributor 契约 |
| 4 | 12 个 provider 的公开 API 文档和 TS 实现足以指导 Rust 实现，无需逐个申请 API key 即可写出 mock 测试。 | high | 单个 provider 实现偏差或无法测试 | 对比 TS 实现和公开 API 文档；单元测试使用 mock HTTP client |
| 5 | TUI 现有工具 chip 渲染机制可以复用，只需新增 `WebSearch` 分支。 | high | 需要新增渲染组件 | 检查 `tui/src/chatwidget/components/tool_chips.rs` 或类似位置 |
